EclipseNDK环境搭建

发表于 2021-01-22 更新于 2021-07-07 分类于 学习

1. 需要下载的内容

  • Eclipse

  • SDK\NDK\ADT

    下载时CTRL+F搜索下载内容asset_img即可，选择符合自己电脑版本

2. 安装下载Eclipse

2.1 进入网址点击Download下载即可

​

​

2.2 安装Eclipse

  • 双击安装程序，出现如下图所示界面选择Eclipse IDE forJava Developers。

  • 选择Java环境以及eclipse安装的位置，点击第一行文件夹图标选择安装Java环境。①为本机环境，②为网上下载。如果本机有环境则可以选择本地环境地址，此处我选择网上安装JRE15.0.1。第二个文件夹图标选择你要下载安装到哪个位置。

  • 出现如下界面选择accept

  • 下载完成后先择launch启动eclipse

  • 启动eclipse后出现如下图所示图标，选择项目存储位置。

3. 给Eclipse安装ADTAndroid开发环境

3.1 安装ADT

  • 启动eclipse后点击help/Install New Software

  • 点击Add之后出现Add Repository界面Name命名为ADT Plugin，点击Archive选择刚刚下载的ADT zip压缩包

  • 选择完成后点击Add，选择需要下载添加的资源点击两次Next之后再点击选择I accept the terms of the license agreements 再点击Finish。等待下载完成即可。

  • **注意 ：**这里要查看右下角下载100%再退出重启，否则会下载失败。

  • 如果安装的很慢，可以安装时可以不选中这个复选框

3.2 Eclipse配置SDK

  • 安装完成后重启Eclipse，点击Windows/Prefrences后则会出现一个Android选项的地方。

  • 打开之后看见Android选项，点击进入配置你的sdk环境，选择好你sdk路径后点击Apply and Close关闭即可。

**注：**如果添加SDK路径如下报错，在你的sdk环境中找到android-sdk-windows\tools\lib目录下plugin.prop文件，把其中的plugin版本更改为与你的ADT版本一致即可。

​

​

3.3 Eclipse配置NDK

  • 同样在Prefrences/Android路径下找到NDK，然后点击找到你的ndk的路径选中，点击Apply and Close结束配置。这样你的NDK就配置成功了。

  • 注：如果在NDK选取路径时报错Not valid NDK directory,则在该路径下创建一个ndk-build文件即可。

ARM汇编学习总结

发表于 2021-01-20 更新于 2021-07-08 分类于 学习

1. Android中ARM学习思路

  • 明白安卓与ARM处理器的关系

    ​ Android嵌入式智能系统基于Linux内核，嵌入式系统Linux是基于ARM平台。Android程序运行基于Linux系统，并且处理器基于ARM。

  • ARM汇编语言程序结构

  • ARM指令

  • ARM处理器的寻址方式

  • ARM指令机器码

2. ARM处理器

​ ARM是一个RISC（精简指令集）处理器。它拥有一套精简的指令集以及比CISC处理器更多的通用寄存器。ARM指令只处理寄存器中的数据，并使用了load/store结构访问存储器，也就是只有load/store指令可以访问存储器。ARM有两种运行模式（可以类比x86的实模式和保护模式），ARM模式和Thumb模式。Thumb指令可以是2或4字节的。

​ 如果要加某个内存地址中保存的值，至少需要三种类型的指令（load指令、加法指令和store指令）。说明：需要使用**load指令将指定地址内存中的值加载到寄存器中，然后使用加法指令增加寄存器中的值，然后用store指令将寄存器中的值写回内存**。

3. RISC(精简指令集)的优劣

  • 优

可以被更快的执行。（原因：RISC处理器通过引入流水线机制，减少每个指令的占用CPU的时钟周期来缩短执行时间）。

注：

时钟周期：时钟周期也成为振荡周期，定义为时钟频率的倒数。时钟周期是计算机中最基本的、最小的时间单位，CPU仅完成一个最基本的动作。时钟周期是一个时间的量。时钟周期表示了SDRAM所能运行的最高频率。更小的时钟周期就意味着更高的工作效率。

流水线：流水线技术是指在程序执行时，多条指令重叠进行操作的一种任务分解术。把一个任务分解成若干顺序执行的子任务，不同的子任务由不同的执行机构来负责执行，而这些执行机构可以同时并行工作。

  • 劣

较少的指令增加了软件（实际上是编译器）的复杂性。

4. RISC/CISC的特点（了解）

4.1 简介

  • CISC

    ​ 最初为了提高操作系统效率，选择向指令系统中添加更多、更复杂的指令来实现，导致指令集越来越大。这种类型的计算机就称为复杂指令集计算机。

  • RISC

    ​ 对指令数据和寻址方式做精简，指令的指令周期相同，采用流水线技术，指令并行执行程度更好，这类是精简指令集计算机。

4.2 CISC主要特点

  • 指令数量多（100-250条）
  • 指令使用频率相差悬殊（最常使用的是较简单的指令）
  • 支持很多种寻址方式（通常为5-20种）
  • 变长的指令（指令的长度不固定，变长的指令增加指令译码电路的复杂性）
  • 指令可以对存储单元中数据直接进行处理（典型的CISC处理器通常都有指令能够直接对内存单元中的数据进行处理，执行速度较慢）

4.3 RISC主要特点

  • 指令数量少（优先选取使用频率高的一些简单指令以及常用指令，避免使用复杂指令）
  • 指令的寻址方式少（只支持寄存器寻址方式、立即数寻址方式以及相对寻址方式）
  • 指令长度固定，指令格式种类少
  • 只提供load/store指令访问存储器
  • 以硬布线逻辑控制为主
  • 单周期指令执行

注：

硬布线：硬布线控制器是早期设计计算机的一种方法。硬布线控制器是将控制部件做成生产专门固定时序控制信号的逻辑电路，产生各种控制信号，因而又称为组合逻辑控制器。这种逻辑电路以使用最少原件和取得最高操作速度为设计目标，因为该逻辑电路又门电路和触发器构成的发杂树型网络，所以称为硬布线控制器。

5. CISC和RISC对比表

指令长度 变化 短、固定

指令格式 复杂 简单

6. ARM和x86/x64的区别

  • ARM中大多数指令可以用于分支跳转的条件判断。
  • Intel的x86/x64系列CPU是小端序的。
  • ARM架构在ARMv3知前是最小端序的之后ARM处理器提供一个配置项，可以通过配置在大端和小端之间切换。

注：

大端模式：是指数据的高字节保存在内存的低地址中，而数据的低字节保存在内存的高地址中，这样的存储模式类似于把数据当作字符串顺序处理：地址由小向大增加，数据从高位往低位放。

小端模式：是指数据的高字节保存在内存的高地址中，而数据的低字节保存在内存的低地址中，这种寸处模式将地址的高低和数据位权有效地结合，高地址部分权值高，低地址部分权值低。

游戏安全学习02

发表于 2021-01-18 更新于 2021-07-06 分类于 学习

1. ARM汇编

​ 目前全部iOS设备和Android设备都使用ARM架构的CPU。学习ARM汇编只需要了解常用的复制、跳转、算数运算、对战操作、内存读写指令和函数调用约定。

2. C、C++语言

​ 游戏引擎的侏罗纪往往会使用C/C++编写，并通过OpenGL ES来绘制界面。目前流行的Coco2d-x引擎可以使用多种语言逻辑编写游戏逻辑,但考虑到执行效率，开发者会优先考虑使用C/C++实现游戏逻辑，且大部分外挂由C/C++来实现相关功能。所以C/C++是极重要的技术基础。（重点关注指针的概念和使用，以及C++虚函数的调用方法）

3. Android开发

​ Android平台的游戏开发还是外挂的开发，绝大部分逻辑代码在Android平台的Native层实现，所以Android开发的最低要求需要：了解程序的生命周期以及Android平台的Native程序开发。

4. 常用的游戏引擎

​ 不同的游戏引擎用来编写游戏侏罗纪的语言、编译器可能不同，需要逆向分析的方法和工具也不一样。Unity 3D游戏大多采用C#开发，游戏的主逻辑大多打包在Assembly-Csharp.dll中，而Cocos2d-x游戏一般采用C++或者Lua开发。

​ 深入了解游戏引擎能帮助快速定位自己关心的游戏逻辑，并实现相关的外挂功能。例如：知道游戏引擎实现碰撞检测的逻辑实现，久能轻松实现穿透类、吸金类和全屏攻击类外挂。

5. 静态分析（IDA）

​ 游戏如果使用了C/C++（Android）、Objective-C（iOS）一般使用IDA静态分析。

​ 在Android平台下，Unity3D游戏会吧其主逻辑模块打包在Assembly-Csharp.dll中，需要使用ILSpy来反编译它，从而分析游戏逻辑。

6. 动态调试（Android调试）

​ 程序加入反静态调试之后可以使用IDA进行动态调试也可以使用GDB（不太成熟，并不方便）。

​ 当动态分析是遇到反调试功能时，可通过静态分析找到反调试代码，并去掉改代码。一般可以动静结合。

7. 必要了解的编程语言

​ ARM汇编、C、C++、Objective-C、Java、smali、Unity 3D（C#）、IL（C#反汇编得到的）、Lua（脚本语言）、Python（脚本语言）。

8. 静态修改

​ 针对不同语言编写的逻辑会有不同的修改方法。

​ C/C++、Objective-C编写的代码：IDA直接修改二进制文件。

​ 针对C#编写的逻辑：会再用ildasm反编译后，修改IL指令，然后通过ilasm编译成DLL文件，替换原安装包对应的DLL文件。

9. 动态修改

​ 静态修改的弊端：有些游戏会对静态文化部做袜子性校验或者签名校验（这样以静态方式绕过校验的成本较高）如果可以动态修改，则可以绕过游戏的静态文件校验。

​ 动态修改游戏逻辑的方式：a.修改进曾中的内存数据b.修改进程中的代码c.调用游戏函数。

​ 实现上述方法的方式：一般需要讲外挂模块注入目标进程，Android中可以利用系统的机制来达到不需要注入模块就能修改数据段，之后再通过直接写进程空间的方式修改数据，可以通过patch代码或者动态Hook的方式修改游戏逻辑。
